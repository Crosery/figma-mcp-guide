两层架构

Figma MCP 由底层原子 API 和上层工作流 Skill 组成。Skill 封装了 Tool 的调用顺序、错误处理和最佳实践。能用 Skill 就用 Skill。

底层 · MCP Tools
  • 由 mcp.figma.com 远程服务器提供
  • 每个工具做一件事：读截图、取元数据、写画布
  • 需要手动传参（fileKey, nodeId）
  • 需要自己编排调用顺序
  • 类比：单个 API endpoint
上层 · Skills
  • 由 figma@claude-plugins-official 插件提供
  • 封装完整工作流 + 最佳实践 + 错误处理
  • 自动编排多个 Tool 的调用
  • 通过 /skill-name 或自然语言触发
  • 类比：封装好的 SDK 方法

T01 get_design_context

核心工具 · 设计转代码的起点

给一个 Figma 节点，返回 React+Tailwind 参考代码 + 截图 + Code Connect 提示 + 资源下载 URL。几乎所有设计转代码工作流的第一步。

参数
nodeId *节点 ID "123:456"（URL 中 - 转 :）
fileKey *从 URL /design/ 后提取
excludeScreenshot省略截图节省 token
disableCodeConnect不走 Code Connect
forceCode大节点强制返回代码
技巧
  • 返回参考代码需适配你的技术栈
  • 节点太大只返回 metadata → forceCode=true
  • 有 Code Connect 时返回你的组件代码
  • branch URL 用 branchKey 作 fileKey
  • 资源 URL 7 天有效，及时下载
  • 默认含截图，除非 excludeScreenshot
04 / 26

T02 get_screenshot

轻量级节点截图

只看设计不生成代码时用它。也是实现后验证视觉一致性的利器。参数只需 nodeId + fileKey。

技巧
  • 比 get_design_context 轻量很多，只返回图片
  • 实现前确认"选对节点没"
  • 实现后对比"代码和设计一不一致"
  • 大页面按 section 分段截图，全页分辨率低
何时用
  • 只看不写码
  • 验证实现结果
  • 逐段检查大页面
  • get_design_context 默认已含截图——仅在上述场景单独用
05 / 26

T03 get_metadata

XML 结构元数据

节点 ID、图层类型、名称、位置、尺寸。没有样式细节，但骨架一目了然。大文件先看它找目标节点。

技巧
  • nodeId="0:1" → 整个页面结构
  • 找到目标 ID → 再调 get_design_context
  • get_design_context 截断时用它找子节点
  • 不能用于 Figma Make 文件
典型流程
get_metadata→get_screenshot→get_design_context

先看骨架 → 截图确认 → 取代码

T04 get_figjam T05 get_variable_defs T06 search_design_system

三个专项读取工具
get_figjam

  • 返回 key 可 import 导入
  • 多用短词并行搜

T07 use_figma

万能写入 · Plugin API JavaScript

在 Figma 文件中执行 JS 代码。创建、编辑、删除任何对象。50KB 限制。

参数
fileKey *目标文件
code *Plugin API JS
description *功能描述
关键规则
  • 必须先 /figma-use skill！
  • return 返回数据 + 所有 node ID
  • 颜色 0-1（不是 0-255）
  • "Semi Bold" 有空格
  • await setCurrentPageAsync() 换页
  • FILL 在 appendChild 之后
  • loadFontAsync 在改文字之前
  • 失败 = 原子未执行，不要盲重试

T08 generate_figma_design T09 generate_diagram T10 create_new_file

三个写入工具
generate_figma_design
  • 网页 URL → Figma 设计文件
  • 支持 localhost + 外部网站
  • 首次不传 outputMode 获取指引
  • 每 5s 轮询 captureId（最多 10 次）
  • 每个 captureId 只能一页
  • 更新已有页面用 use_figma
generate_diagram
  • Mermaid.js → FigJam 图表
  • 支持流程/时序/状态/甘特图
  • 不支持类图/ER/时间线
  • 文本加引号，不用 emoji
  • 返回 URL 必须展示给用户
create_new_file
  • 创建空白 Design / FigJam 文件
  • 需 planKey（先调 whoami）
  • 返回 fileKey + URL
whoami→create→use_figma

S01 /figma-implement-design

Figma 设计 → 项目代码（1:1 复刻）

最常用 Skill。给 Figma 链接，走 7 步标准流程生成生产级代码。自动适配技术栈、复用组件、下载资源、验证一致性。

7 步流程
  • 1. 解析 URL → fileKey + nodeId
  • 2. get_design_context → 代码 + 截图
  • 3. get_screenshot → 视觉基准
  • 4. 下载资源（图片 / SVG）
  • 5. 翻译为项目技术栈
  • 6. 追求 1:1 视觉还原
  • 7. Checklist 验证
原则
  • 截断 → 自动 get_metadata 分批取
  • 优先复用已有组件
  • Token → Token 映射，不硬编码
  • 偏差记录在注释说明
  • 验证：布局/字体/颜色/交互/响应式/a11y

S02 /figma-use

use_figma 的强制前置 Skill

每次调 use_figma 前必须加载。14 条规则 + 错误恢复 + 飞行前清单 + 10 份参考文档索引。不加载 = 各种难调的 bug。

加载了什么
  • 14 条关键规则
  • 页面上下文重置规则
  • 错误恢复：失败 = 原子性无残留
  • 增量工作流：一步一验证
  • 飞行前 14 项必查
  • 10 份参考文档索引
速查
  • return 数据 + node ID
  • 禁 figma.notify() / console.log()
  • 颜色 {r:1,g:0,b:0} = 红
  • await 每个 Promise
  • FILL/HUG 在 appendChild 之后
  • loadFontAsync 在改文字之前

S03 /figma-generate-design

代码 / 描述 → Figma 画布

将代码或描述转为 Figma 设计稿。核心：不画裸矩形——发现并复用设计系统的组件、变量、样式，逐段组装。

6 步
  • 1. 理解画面，列各段和组件
  • 2. search_design_system 找资源
  • 3. 创建 Wrapper Frame
  • 4. 逐段构建（每段一次 use_figma）
  • 5. 每段 get_screenshot 验证
  • 6. 更新已有：定位→修改→验证
原则
  • 不硬编码 hex/px — 用变量绑定
  • importComponentByKeyAsync 复用
  • setBoundVariable 绑 Token
  • 文字用 setProperties() 覆盖
  • 不要建顶层节点再移入 wrapper

S04 /figma-generate-library

从代码构建完整 Figma 设计系统

大型工程 Skill — 20~100+ 次 use_figma，5 阶段，每阶段用户确认检查点。

5 阶段
  • Phase 0 · Discovery — 分析+锁定范围
  • Phase 1 · Foundations — 变量/模式/别名/样式
  • Phase 2 · File Structure — 页面+文档
  • Phase 3 · Components — 逐个原子→分子
  • Phase 4 · Integration QA — 审计
规则
  • 最小单元 + 验证，不要一步到位
  • Variable 在 Component 之前
  • 每个 Variable 设 scopes
  • 语义层 alias 到 Primitive
  • Variant > 30 → 拆分
  • 状态持久化到 /tmp/dsb-state.json
  • 每阶段用户确认才继续

S05 /figma-code-connect

创建 .figma.js Code Connect 模板

将 Figma 组件映射到代码。创建 parserless 模板让 Dev Mode 显示你的组件代码。需 Org/Enterprise 计划。

6 步
  • 1. 解析 URL
  • 2. suggestions 发现未映射组件
  • 3. context 获取属性定义
  • 4. 代码库找匹配组件
  • 5. 创建 .figma.js 模板
  • 6. 验证属性覆盖
属性映射
TEXTgetString('Name')
BOOLEANgetBoolean('Name', {...})
VARIANTgetEnum('Name', {...})
INSTANCE_SWAPgetInstanceSwap('Name')
子图层findInstance('Layer')
文本层findText('Name').textContent

S06 /figma-create-design-system-rules

生成项目设计系统规则

分析代码库，生成规则保存到 CLAUDE.md / AGENTS.md / .cursor/rules/。后续 Figma 转代码自动遵循规范。

流程
  • 1. 调用 create_design_system_rules 获取模板
  • 2. 分析代码库结构
  • 3. 生成专属规则
  • 4. 保存到规则文件
  • 5. 简单组件测试
覆盖范围
  • 组件：位置 / 命名 / 导出
  • 样式：CSS 方案 / Token / 禁硬编码
  • 流程：必须先 get_design_context
  • 资源：localhost 直接用
  • 规范：路由 / 状态 / import
项目初始化跑一次。给 AI 写了份"新人入职手册"。

S07 /figma-to-localui

Figma → Vue 组件 CSS 变量同步

自定义 Skill — Figma 选区实时读取样式，和 Vue CSS 变量对比，精准更新 variables.css。支持双 MCP。

5 步
  • 1. get_selection + get_node_info
  • 2. 提取样式（尺寸/圆角/颜色/字体）
  • 3. 找对应 Vue 组件和 CSS 变量
  • 4. 对比 → 修改 variables.css
  • 5. 资源替换（SVG/背景图）
双 MCP 策略
场景MCP
日常样式对比本地插件（无限/实时）
下载资源图官方远程（7 天 URL）
批量导出 SVG本地 get_svg
  • 优先改 CSS 变量非模板
  • 局部覆盖全局 Token
  • SVG fill → currentColor
